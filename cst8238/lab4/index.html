Lab 4 - CSS and Multimedia

What are the main benefits of using CSS?

CSS allows for the separation of web design and web programming. Web designers can write a cascading style-sheet independently from the actual implementation of the website, and this sheet can then be reused and shared between pages and projects. This allows for a consistent style, including color palette, typeface, and page layout.

What are the different parts of a CSS style rule?

There are two main parts to a CSS style rule. First, a selector, which is explained in further detail later on, and one or more rules. These rules can change the size, shape, color, of an element, and change many other properties as well. In the example below, .someclass is the selector, and the rules are contained within the curly braces.

Ex: .someclass { background-color: red; font-family: 'Times New Roman'; }

What are CSS selectors?

Using CSS selectors, the web designer can select precisely which rules should be applied to which elements. For example, styles can be applied to all instances of a tag by simply using the tag name in the style sheet. Examples:

body { color: red; } will set all text in the body to red, as well as all children of body, unless another rule overwrites the style. This selector will apply to all <body> elements

.someClass { ... } This is a class selector. All elements with "someClass" in their class attribute will be selected.

#someID { ... } This is an ID selector. All elements with "someID" in their id attribute will be selected.

These rules can be "chained" together, allowing for precise element styling. This is called a "contextual selector", and will match when an element is a descendant of some ancestor element. The matched elements do not need to be immediate descendants to be matched.

Ex. div ul li a.item {}

The rule above would match all elements of type "a" with a class of "item", with some ancestor li, with some ancestor ul, with some ancestor div.

Ex: div.content ul li, div.content ol li { ... }

Browser Cascade Principles

In larger projects, it is common for there to be many stylesheets, some of which either deliberately or accidentaly overwrite certain properties. CSS has rules for handling these scenarios. Primarily, stylesheets are cascading, meaning that when two rules are applied to a single element, the rule loaded last is applied. These rules could be in the same file, or a rule could be overwritten when a new stylesheet is loaded by the browser. The second deciding factor is specificity. In CSS, an element selector is less specific than a class selector. An element selector will select any matching elements on a page, whereas a class selector is more specific.

Example:
.heading {color: yellow;}
h1 {color: blue;}
...
<h1 class="heading">Title</h1>
In this case, the title would appear yellow, as a class selector is more specific.
