Basics of Web Technology

  • Why do we need a Uniform Resource Locator (URL)?
    • A url is used to point to a specific resource on a web server.
    • ex. https://github.com/jlcarveth/web.git
  • A URL is made up of the following parts:
    • A protocol (https://, ftp://)
    • A hostname (github.com)
    • A path (/jlcarveth/web.git)
  • What is the difference between a static and dynamic web site?
    • A static web site consists of raw HTML files which are not altered when delivered to the user. All visitors of the site will see the same content.
    • A dynamic web site displays content based on application state, user contents stored in cookies or LocalStorage, URL parameters, or many other factors. Dynamic webpages can display different content based on who is accessing the website. Such a web site could be as simple as an HTML file with Javascript to manipulate the DOM, or could consist of a web server rendering and delivering custom HTML and CSS to each individual user.
  • What is DNS and Why do we need it?
    • The Domain Name System is essentially an phone book for the internet. Similarily to a phone book, a DNS record has a name (domain name) and a phone number (ip address) which is used to contact the 'person'.
    • We use the DNS as it is far easier to remember a domain name (google.com) than an IP address (172.217.0.238)
    • DNS resolution is the process of contacting a DNS server which will return the desired IP address.
    • Some top-level domains (TLDs) include:
      • Generic TLDs:
        • .com
        • .gov
        • .org
        • .edu
      • Country-code TLDs:
        • .ca
        • .tv
        • .ru
  • What is the purpose of the HTTP protocol?
    • The purpose of the HTTP protocol is to standardize the fetching of documents on the web. It defines requests (GET, POST, PUT, PATCH, DELETE) to modify and retrieve documents, and standardizes responses web servers should provide.
  • Define a web server:
    • A web server hosts content for one or more websites, and is responsible for responding to requests and serving content accordingly.
