    Getting Started With Kotlin

    JLCarveth
    2017-10-20
    Java, Kotlin

    I really enjoy Java. It’s a fairly type safe language, helping to create cleaner, less bug-prone code. It’s a very strong object oriented language, which allows for easier design and implementation of software engineering models and concepts. Java is also a very old language, it’s first version dating back twenty-two years. Java’s maturity definitely has its advantages, the many years allowing for an incredibly large community, and an incredibly vast selection of third-party libraries. But Java’s foundation is built on outdated concepts. This is where Kotlin comes in.

    If you’ve been keeping up with Android Development at all, you’ve probably heard a thing or two about Kotlin. Kotlin is essentially a modernized version of Java, fixing old Java isses with plenty of new features, all while compiling into Java byte-code. Some of these features and fixes include:

    • No more primitive types (now everything is an object)
    • It’s concise.
    • It’s completely inter-operable with Java
      • Kotlin can use all of Java’s libraries, as well as most third party Java libraries.
      • This includes Android. You can use Kotlin and Java classes in the same project.
      • Kotlin in Android requires far less boilerplate code compared to an equivalent Java class. The next blog post will cover this in greater detail.

    Now, let’s jump into writing Kotlin code!

    Getting Started With Kotlin’s Syntax

    If you’ve gotten this far, I’m glad I could convince you to give Kotlin a shot. In order to start Kotlin, I would recommend installing IntelliJ IDEA. It’s made by the same people who develop Kotlin, and thus has great support for the language out of the box. Kotlin can also be written in Eclipse, a terminal, or even in their online editor.

    Now that you’ve (hopefully) gotten your development method of choice set-up, I’ll cover some of Kotlin’s basics. All of this code is available in the GitHub repository for this post.

    Where to Implement Kotlin?

    Now you’ve seen some of what Kotlin can do, so when would you use Kotlin? Kotlin is great for just about anything, from client side, server side, web, and Android! And because Kotlin works with Java so well, you can start writing new classes in your existing Java project and have the program still work perfectly. Or you could work at converting a project from Java to Kotlin one class at a time. Kotlin leaves the choice up to you.
